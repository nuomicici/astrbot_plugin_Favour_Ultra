## 更新日志

### v4.0.0（2026-06-01）
*   **架构重构**：
    *   新增 `config_manager.py`，自行管理配置文件，不再依赖框架配置逻辑。
    *   新增 `pages/` WebUI 可视化配置面板，所有参数均可在线修改。
    *   首次加载自动检测并迁移旧版框架配置，无需手动迁移。
    *   Prompt 注入方式改为 system_prompt（固定规则） + extra_user_content_parts（动态数据）双通道分离，不影响缓存。
    *   好感度标签正则表达式升级，新增英文 fallback（`[Favour increased:X]` 等）和中文容错（`[好-感度 持平]` 等变形均可识别）。
*   **好感度分级系统**：新增 `favour_levels` 配置，支持自定义 7 级等级名称、数值区间和行为描述。Prompt 仅注入当前用户所在等级规则，减少 Token 消耗。更新后旧有规则失效，请务必检查一次！
*   **好感度衰减**：支持线性 / 分级两种衰减模式。无互动超过设定天数后自动降低好感度，后台异步调度，无需人工干预。数据库新增 `last_interaction` 字段。
*   **主动搭话**：按好感度区间配置触发概率，支持自定义时间窗口和检查间隔，可配置 LLM 搭话提示词。
*   **查询权限**：新增 `query_permission` 配置，可分别控制群聊/私聊普通用户的好感度查询权限。
*   **自动拉黑**：好感度达到最低值时自动拉黑用户，不再监听其消息。
*   **LLM 自主关系管理**：
    *   新增 `[主动解除关系:目标用户ID]` / `[主动解除关系:目标用户ID:关系名称]` 标签，LLM 可在好感暴跌、背叛/冒犯等场景主动断交。解除后好感度不变，仅清除关系。同目标全程限 2 次（在历史对话上限次数内）。
    *   新增 `[主动确认关系:目标用户ID:关系名称:排他性]` 标签，LLM 可在情感自然发展时主动建立关系。极度克制的使用限制，仅限真挚场景。
    *   关系标签升级为新格式 `[用户申请确认关系:目标用户ID:关系名称:同意:排他性]`，旧 3 字段格式向下兼容。省略目标用户ID 时自动指向发送者。
    *   输出链自动过滤全部 4 种关系标签，用户不可见。
*   **Prompt 中文化**：所有注入指令从英文/混杂改为纯中文描述，适配中文模型。
*   **Prompt 压缩**：system_prompt 部分大幅精简，去除冗余措辞和兼容性描述，标签格式定义原样保留。
